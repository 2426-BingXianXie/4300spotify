# Spotify Graph Music Recommender (DS 4300)

A general-purpose song recommendation system built on **Neo4j** and the
**Graph Data Science** plugin. It builds a kNN audio-feature similarity
graph over a stratified sample of the Kaggle Spotify dataset, then runs
**personalized PageRank** seeded on a user's liked songs to recommend
five new tracks.

Tested seed set: **The Strokes + Regina Spektor** (Prof. Rachlin's
favorites). Strokes and Spektor songs are excluded from the final
recommendations.

See [graph_music_rec.pdf](graph_music_rec.pdf) for the assignment spec.

## Similarity rule

Two songs `u` and `v` are connected by a `SIMILAR_TO` edge **iff `v` is
among the K (= 10 by default) nearest neighbors of `u` in standardized
audio-feature space, using Euclidean distance**. Edges are symmetric and
weighted by `score = 1 / (1 + distance)` (higher = more similar).

## Quick start

```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
cp .env.example .env   # fill in NEO4J_URI / NEO4J_USER / NEO4J_PASSWORD
```

Start a local Neo4j 5.x with the **Graph Data Science** plugin installed
and the database running on the URI in `.env`. Then:

```bash
python -m src.preprocess     # data/sample.csv
python -m src.similarity     # data/similarity.csv
python -m src.build_graph    # load schema + nodes + edges into Neo4j
python -m src.recommend      # prints top-5 recommendations + metrics
```

`recommend.py` prints `N`, `E`, and density (the poster numbers) and a
table of recommended tracks with their artists, album, genres, and PPR
score.


## Results (latest run, SEED=4300, SAMPLE_SIZE=3000, K=10)

- **Nodes**: 3045 `Song`, 3252 `Artist`, 114 `Genre`.
- **Edges**: 42,384 directed `SIMILAR_TO` (= 21,192 unique undirected pairs).
- **Graph density** (undirected over Song nodes):
  `(2 * 21192) / (3045 * 3044) ≈ 0.00457`.
- **Sample composition**: 43 unique Strokes tracks, 3 Regina Spektor tracks
  (after dedupe on `(track_name, artists)`).
